PackageÔÉÅ

XML tag

<AR-PACKAGE>

Module

autosar.package

Inherits

An AUTOSAR package is a container for elements. Elements are almost everything else that are not packages. In addition to elements each package can also contain zero or more sub-packages.

UsageÔÉÅ

import autosar

ws = autosar.workspace("4.2.2")
package = ws.createPackage('ComponentTypes', role = 'ComponentType')

createCompuMethodRationalÔÉÅ

Package.createCompuMethodRational(name, offset, scaling[, lowerLimit = None][, upperLimit = None][, lowerLimitType = 'CLOSED'][, upperLimitType = 'CLOSED'][, unit = None][, defaultValue = None][, label = 'SCALING'][, forceFloat = False][, useIntToPhys=True][, usePhysToInt = False][, category = 'LINEAR'][, adminData = None])ÔÉÅ

Creates a new CompuMethod containing a CompuScaleElement (offset+scaling) and adds it to the package.

Parameters

  • name (str) ‚Äì ShortName of the new CompuMethod.

  • offset (int,float) ‚Äì Offset.

  • scaling (int, float, rational number) ‚Äì Scaling factor.

  • lowerLimit (int) ‚Äì Lower limit of the CompuScaleElement.

  • upperLimit (int) ‚Äì Upper limit of the CompuScaleElement.

  • lowerLimitType (str) ‚Äì Interval type of lowerLimit (‚ÄúOPEN‚Äù or ‚ÄúCLOSED‚Äù). Only applies when lowerLimit is not None.

  • upperLimitType (str) ‚Äì Interval type of upperLimit (‚ÄúOPEN‚Äù or ‚ÄúCLOSED‚Äù). Only applies when upperLimit is not None.

  • unit (str) ‚Äì Optional unit name (requires package role ‚ÄòUnit‚Äô to be setup).

  • defaultValue (None, int, float, str) ‚Äì Optional default value.

  • label (str) ‚Äì Label of the CompuScaleElement

  • forceFloat (bool) ‚Äì When False it sets the scaling numerator/denominator to closest possible rational number. When True it forces numerator to float and sets denominator to 1.

  • useIntToPhys (bool) ‚Äì When True, creates an internal to physical Computation. Cannot be True when usePhysToInt is also True.

  • usePhysToInt (bool) ‚Äì When True, creates a physical to internal Computation. Cannot be True when useIntToPhys is also True.

  • category (str) ‚Äì Category string used for the new CompuMethod.

  • adminData (None, AdminData.) ‚Äì Optional AdminData.

Return type

CompuMethod

This method by default creates an internal to physical computation, use the method createCompuMethodRationalPhys to create a physical to internal computation. Some AUTOSAR toolchains does not work when numerator and denominator is a rational number. Use the option forceFloat to force the scaling factor to be stored as a floating point number.

createCompuMethodRationalPhysÔÉÅ

Convenience method that calls createCompuMethodRational but creates physical to internal computation instead.

createImplementationDataTypeÔÉÅ

Package.createImplementationDataType(name, baseTypeRef[, lowerLimit = None][, upperLimit = None][, valueTable = None][, bitmask = None][, offset = None][, scaling = None][, unit = None][, forceFloat = False][, dataConstraint = ''][, swCalibrationAccess = ''][, typeEmitter = None][, lowerLimitType = None][, upperLimitType = None][, category='VALUE'][, adminData = None])ÔÉÅ

Creates a new implementation data type that is tied directly to a base type.

Parameters

  • name (str) ‚Äì ShortName of the datatype

  • baseTypeRef (str) ‚Äì Reference to existing base type

  • lowerLimit (int) ‚Äì Optional lower limit.

  • upperLimit (int) ‚Äì Optional upper limit.

  • valueTable ‚Äì Optional valueTable. See valueTable for details.

  • bitmask ‚Äì Optional bitmask. See bitmask for details.

  • lowerLimitType (str) ‚Äì Selects lowerLimitType (‚ÄúOPEN‚Äù or ‚ÄúCLOSED‚Äù) if lowerLimit is used. Value None defaults to ‚ÄúCLOSED‚Äù.

  • adminData (None, AdminData.) ‚Äì Optional AdminData.

ExampleÔÉÅ

import autosar

#Setup
ws = autosar.workspace(version="4.2.2")
package=ws.createPackage('DataTypes', role='DataType')
package.createSubPackage('CompuMethods', role='CompuMethod')
package.createSubPackage('DataConstrs', role='DataConstraint')

baseTypes=package.createSubPackage('BaseTypes')
baseTypes.createSwBaseType('uint8', 8, nativeDeclaration='uint8')
baseTypes.createSwBaseType('uint16', 16, nativeDeclaration='uint16')
baseTypes.createSwBaseType('uint32', 32, nativeDeclaration='uint32')

implTypes = package.createSubPackage('ImplementationTypes')
implTypes.createImplementationDataType('uint8', lowerLimit=0, upperLimit=255, baseTypeRef='/DataTypes/BaseTypes/uint8', typeEmitter='Platform_Type')
implTypes.createImplementationDataType('uint16', lowerLimit=0, upperLimit=65535, baseTypeRef='/DataTypes/BaseTypes/uint16', typeEmitter='Platform_Type')
implTypes.createImplementationDataType('uint32', lowerLimit=0, upperLimit=4294967295, baseTypeRef='/DataTypes/BaseTypes/uint32', typeEmitter='Platform_Type')

#Export as XML
ws.saveXML('DataTypes.arxml')

createImplementationDataTypeRefÔÉÅ

Package.createImplementationDataTypeRef(name, implementationTypeRef[, lowerLimit = None][, upperLimit = None][, valueTable = None][, bitmask = None][, offset = None][, scaling = None][, unit = None][, forceFloat = False][, dataConstraint = ''][, swCalibrationAccess = ''][, typeEmitter = None][, lowerLimitType = None][, upperLimitType = None][, category = 'TYPE_REFERENCE'][, adminData = None])ÔÉÅ

Creates a new implementation data type that is a reference to another implementation type. This is similar in concept to a typedef in the C programming language

Parameters

  • name (str) ‚Äì ShortName of the datatype

  • implementationTypeRef (str) ‚Äì Reference to existing ImplementationDataType

  • lowerLimit (int) ‚Äì Optional lower limit.

  • upperLimit (int) ‚Äì Optional upper limit.

  • valueTable ‚Äì Optional valueTable. See valueTable for details.

  • bitmask ‚Äì Optional bitmask. See bitmask for details.

  • offset (int) ‚Äì Optional offset (for CompuMethod). When this is used, scaling also has to be set.

  • scaling (int or float) ‚Äì Optional scaling factor (for CompuMethod).

  • unit (str) ‚Äì Optional unit.

  • forceFloat (bool) ‚Äì When False it sets the scaling numerator/denominator to closest possible rational number. When True it forces numerator to float type and sets denominator to 1.

  • dataConstraint (str) ‚Äì Optional name or reference to existing DataConstraint element. Value None prevents dataConstraint from being created while empty-string automatically attempts to create a DataConstraint element.

  • swCalibrationAccess (str) ‚Äì Optional calibration access

  • typeEmitter (str) ‚Äì Optional type emitter string.

  • lowerLimitType (str) ‚Äì Selects lowerLimitType (‚ÄúOPEN‚Äù or ‚ÄúCLOSED‚Äù) if lowerLimit is used. Value None defaults to ‚ÄúCLOSED‚Äù.

  • upperLimitType (str) ‚Äì Selects upperLimitType (‚ÄúOPEN‚Äù or ‚ÄúCLOSED‚Äù) if upperLimit is used. Value None defaults to ‚ÄúCLOSED‚Äù.

  • category (str) ‚Äì Category string.

  • adminData (None, AdminData.) ‚Äì Optional AdminData.

Return type

ImplementationDataType

ExampleÔÉÅ

import autosar

#Setup
ws = autosar.workspace(version="4.2.2")
package=ws.createPackage('DataTypes', role='DataType')
package.createSubPackage('CompuMethods', role='CompuMethod')
package.createSubPackage('DataConstrs', role='DataConstraint')
baseTypes=package.createSubPackage('BaseTypes')
baseTypes.createSwBaseType('uint8', 8, nativeDeclaration='uint8')
baseTypes.createSwBaseType('uint16', 16, nativeDeclaration='uint16')
implTypes = package.createSubPackage('ImplementationTypes')
implTypes.createImplementationDataType('uint8', lowerLimit=0, upperLimit=255, baseTypeRef='/DataTypes/BaseTypes/uint8', typeEmitter='Platform_Type')
implTypes.createImplementationDataType('uint16', lowerLimit=0, upperLimit=65535, baseTypeRef='/DataTypes/BaseTypes/uint16', typeEmitter='Platform_Type')




#Create implementation data type with offset and scaling
implTypes.createImplementationDataTypeRef('VehicleSpeed_T',
                                            implementationTypeRef = '/DataTypes/ImplementationTypes/uint16',
                                            lowerLimit = 0,
                                            upperLimit = 65535,
                                            offset = 0,
                                            scaling = 1/64,
                                            forceFloat = True)

#Create implementation data type with enumeration table (also known as value table)
implTypes.createImplementationDataTypeRef('OffOn_T', implementationTypeRef = '/DataTypes/ImplementationTypes/uint8',
                                          valueTable = ['OffOn_Off',
                                                        'OffOn_On',
                                                        'OffOn_Error',
                                                        'OffOn_NotAvailable'
                                                        ])

#Export as XML
ws.saveXML('DataTypes.arxml')

valueTableÔÉÅ

The valueTable parameter accepts three different formats:

  1. List of strings

  2. List of 2-tuples

  3. List of 3-tuples

List of stringsÔÉÅ

List of text values. LowerLimit starts at 0 and is automatically incremented by 1 for each element in the list. The UpperLimit follows LowerLimit for each element.

List of 2-tuplesÔÉÅ

The first element of the tuple is used as both lowerLimit and upperLimit (int). The second element is the text value (str).

List of 3-tuplesÔÉÅ

The first element of the tuple is lowerLimit (int). The Second element is the upperLimit (int). The third element is the text value (str).

bitmaskÔÉÅ

The bitmask parameter is a list of 2-tuples. The first element of the tuple is the bitmask (int). The second elementof is the symbol (str).
